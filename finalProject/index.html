// philosophy

code lives in
the filesystem.

tree reimagines what source code looks like. Your program is a folder. Subfolders are statements. Files are values. The numbering determines execution order. No text editor required — just your OS.

📁

Folders are Statements

A folder named 1.for is a for loop. 2.if is a conditional. Nesting defines scope.

📄

Files are Values

A file named 1.i is the variable i. A file named 2.42 is the literal 42.

🔢

Numbers are Order

The prefix number on each file or folder determines execution order. 1. runs before 2. always.

⚡

Python Interpreter

A lightweight Python script walks your directory, parses it with TextX, and executes it immediately.

// reference

language syntax

Every construct follows the pattern N.keyword\ as a folder containing numbered children.

constructfolder namedescription
assignassign\ → 1.var, 2.valueAssign a value to a variable
forfor\ → 1.assign, 2.condition, 3.assign, 4.do\For loop
ifif\ → 1.condition, 2.do\, [elif\], [else\]Conditional branch
printprint\ → 1.valuePrint a value or expression
addadd\ → 1.left, 2.rightAddition or string concat
subsub\ → 1.left, 2.rightSubtraction
mulmul\ → 1.left, 2.rightMultiplication
divdiv\ → 1.left, 2.rightDivision
modmod\ → 1.left, 2.rightModulo
equalsequals\ → 1.left, 2.rightEquality check
lessthanlessthan\ → 1.left, 2.rightLess than
greaterthangreaterthan\ → 1.left, 2.rightGreater than

// getting started

run it yourself

Clone the repo, install the one dependency, and point the interpreter at any .tree folder.

$git clone https://github.com/is9belle/tree
$cd tree
$pip install textx
$py tree.py FizzBuzz.tree
